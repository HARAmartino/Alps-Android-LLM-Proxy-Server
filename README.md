# Alps-Android-LLM-Proxy-Server
目的: 古いAndroidスマートフォン（Android 8.0+）を、ローカルネットワークおよびインターネット経由でアクセス可能な HTTPS対応・ストリーミング転送可能な汎用LLM APIプロキシサーバー として構築する。 
ターゲットユーザー: ローカルLLM利用者、外出先からも自前インフラでAPIを利用したい開発者、他サービスと連携させるバックエンド運用者。 主要制約: メモリ制限（2GB RAM想定）、バッテリー最適化回避、動的ネットワーク環境（自宅/外出先）への対応、Android OSのバックグラウンド制約。

アーキテクチャ方針:
完全透過型プロキシ（ボディ変換なし）を維持し、低メモリ・低CPU負荷を実現。
デュアルモードネットワーク: ユーザーが「ポート開放モード」または「トンネリングモード」を選択可能。
モダンAndroid準拠: DataStore、ViewModel + StateFlow、Android 14対応ForegroundServiceを採用。
